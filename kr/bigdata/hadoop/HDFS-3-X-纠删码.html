# HDFS 3.X 纠删码

HDFS 默认使用三副本机制来保证存储的高容错性，也带来了高昂的存储成本；HDFS 3.X 引入了纠删码实现与多副本机制相同的容错能力，但能够大幅度降低存储成本。

# 纠删码(EC)

Erasure Coding：纠删码是一种编码技术，在磁盘冗余阵列(RAID)中通过条带化技术实现纠删码。

条带化技术：一种自动将 I/O 的负载均衡到多个物理磁盘上的技术，原理是将一块连续数据分片存储到不同磁盘上，当多进程同时访问数据的不同分片时不会造成磁盘冲突（当多进程同时访问同一磁盘时，可能会出现磁盘冲突），而且在需要对这种数据进行顺序访问时可以获得最大程度上的 I/O 并行能力，从而获得非常好的性能。

在 HDFS 中，把连续数据的分片称为条带化单元，对于原始数据单元的每个条带化单元，都会计算并存储一定数量的奇偶检验单元（编码），可以通过基于剩余数据和奇偶校验单元的解码计算来恢复任何条带化单元上的错误。

# EC 应用场景

将 EC 技术集成进 HDFS 可以提高存储效率，同时仍提供与传统的基于副本的 HDFS 部署类似的数据持久性。但是 EC 在编码过程及数据重建期间会占用大量 CPU 资源，并且数据大部分是执行远程读取，所以还会有大量的网络开销。故：传统上已将其用于存储较冷，访问频率较低的数据

  • 对于 CPU 资源紧张且存储成本较低的情况下，可以采用副本机制存储数据
  • 对于 CPU 资源有剩余且存储成本较高的情况下，可以采用 EC 机制存储数据
