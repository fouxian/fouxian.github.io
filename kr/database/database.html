# 基础与核心

  • 元组：元组是关系模型的基本概念，关系是一张表，表中每行就是一个元组，每列就是一个属性
  • 候选码（码）：关系中的属性/属性组能唯一标识一个元组（元组函数依赖于候选码），码中属性非空
  • 主码：主键，主码是从多个候选码中选出来唯一一个
  • 外码：外键，关系中一个属性是另一关系中的主码，则这个属性为外码
  • 主属性：候选码中的属性
  • 非主属性：不包含在任何一候选码中的属性（非主属性函数依赖于码）
  • 函数依赖：在 X 值确定情况下，必定能确定 Y 的值，则 Y 函数依赖于 X，X → Y
  • 完全函数依赖：若 X → Y，且对于 X 的任一真子集 Z，Z → Y 不成立，则 Y 完全函数依赖于 X
  • 部分函数依赖：若 Y 函数依赖于 X，且 Y 不完全函数依赖于 X，则 Y 部分函数依赖于 X
  • 传递函数依赖：若 Z 函数依赖于 Y，Y 函数依赖于 X，Y 不包含于 X，且 X 不函数依赖于 Y，则 Z 传递函数依赖于 X

# ER

ER：实体关系模型，一种提供了实体、属性和联系的方法，用来描述现实世界的概念模型

  • 实体：实际问题中客观存在的并且可以相互区别的事物，实体是现实世界中的对象
  • 属性：实体所具有的某一个特性，可用来描述实体
  • 键：在描述实体的所有属性中，可以唯一标识实体的属性，唯一且非空
  • 类型：具有相同特征和性质的多个实体的共同抽象
  • 联系：实体间的联系通常有3种类型：一对一、一对多、多对多

# 三范式（NF）

范式（NF）：表示一个关系内部各属性间联系的合理化程度，简单说一张数据表的表结构所符合的某种设计标准的级别，符合高一级范式的设计，必定符合低一级范式

  • 1NF：符合 1NF 的关系中每个属性都不可再分，是所有关系型数据库的最基本要求，第一范式会造成数据冗余问题，需要规范化
  • 2NF：在 1NF 基础上，消除了非主属性对于码的部分函数依赖，消除过程称为模式分解
  • 3NF：在 2NF 基础上，消除了非主属性对于码的传递函数依赖
