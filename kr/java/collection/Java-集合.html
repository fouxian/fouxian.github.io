# Java 集合

一般情况下，通过声明一个固定容量的数组来存储相同类型的元素，此时类型、容量固定，元素有序可重复，而集合能够提高数据存储的灵活性。

List/Queue：有序可重复；Set：无序不可重复；Map：键值对，key：无序不可重复，value：无序可重复

# List：Vector/Stack（过时少用）

  • Vector ：实现 List 的动态数组，其中的操作是线程安全的（由 synchronized 修饰）
  • Stack：继承 Vector，是后进先出（LIFO）的栈，也是线程安全

# List：ArrayList（扩容机制）

public class ArrayList<E> extends AbstractList<E> implements List<E>, RandomAccess, Cloneable, java.io.Serializable {
	private static final int DEFAULT_CAPACITY = 10; // 默认容量
	private static final Object[] EMPTY_ELEMENTDATA = {}; // 容量为零的空数组
	private static final Object[] DEFAULTCAPACITY_EMPTY_ELEMENTDATA = {}; // 空数组，添加第一个元素时，扩容为默认容量
	transient Object[] elementData; // 数组
	private int size; // 数组元素个数，非容量，size() 方法返回值
	// 经常使用的无参构造使用的是 DEFAULTCAPACITY_EMPTY_ELEMENTDATA 实例
	public ArrayList() {this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;}
	public void trimToSize() { // 清除多余空间
		modCount++;
		if (size < elementData.length) {
			elementData = (size == 0) ? EMPTY_ELEMENTDATA : Arrays.copyOf(elementData, size);
		}
	}

	public boolean add(E e) { // 数组末尾添加元素
		ensureCapacityInternal(size + 1); // 扩容计算与判断
		elementData[size++] = e;
		return true;
	}
	public void add(int index, E element) { // 指定位置添加元素
		rangeCheckForAdd(index); // 边界检查，0 <= index <= size
		ensureCapacityInternal(size + 1); // 扩容计算与判断
		System.arraycopy(elementData, index, elementData, index + 1, size - index); // 将指定位置开始后的元素后移一位
		elementData[index] = element;
		size++;
	}
	public boolean addAll(Collection<? extends E> c) {
		Object[] a = c.toArray();
		int numNew = a.length;
		ensureCapacityInternal(size + numNew);
		System.arraycopy(a, 0, elementData, size, numNew);
		size += numNew;
		return numNew != 0;
	}
	public boolean addAll(int index, Collection<? extends E> c) {
		rangeCheckForAdd(index);
		Object[] a = c.toArray();
		int numNew = a.length;
		ensureCapacityInternal(size + numNew);
		int numMoved = size - index;
		if (numMoved > 0) System.arraycopy(elementData, index, elementData, index + numNew, numMoved);
		System.arraycopy(a, 0, elementData, index, numNew);
		size += numNew;
		return numNew != 0;
	}
	/**
	 * 扩容机制
	 */
	private void ensureCapacityInternal(int minCapacity) {ensureExplicitCapacity(calculateCapacity(elementData, minCapacity));}
	
	// 计算扩容容量
	private static int calculateCapacity(Object[] elementData, int minCapacity) {
		if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
			// 添加前十个元素时，minCapacity = [1, 10] ，最大扩容为 10
			return Math.max(DEFAULT_CAPACITY, minCapacity);
		}
		return minCapacity;
	}
	// 判断是否需要扩容
	private void ensureExplicitCapacity(int minCapacity) {
		modCount++;
		if (minCapacity - elementData.length > 0) 
			// 当添加第一个元素时，扩容(10)，当添加第 11 个元素时，再次扩容(15，每次1.5倍左右) ... 依次类推
			// 当每次添加元素超过原有容量时，进行扩容为原有容量的 1.5 倍左右
			grow(minCapacity); // 扩容
	}
	private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8; // 最大数组大小，防止 OOM
	private void grow(int minCapacity) { // 扩容
		int oldCapacity = elementData.length; // 旧容量
		// oldCapacity 右移一位，等价于 oldCapacity / 2；位运算快于整除运算
		int newCapacity = oldCapacity + (oldCapacity >> 1); // 新容量，扩容 1.5 倍
		if (newCapacity - minCapacity < 0) // 与最小容量比较，每次扩容为原有容量的 1.5 倍左右
			newCapacity = minCapacity;
		// 当 newCapacity > MAX_ARRAY_SIZE && minCapacity > MAX_ARRAY_SIZE，则容量为 Integer.MAX_VALUE
		// 当 newCapacity > MAX_ARRAY_SIZE && minCapacity <= MAX_ARRAY_SIZE，则容量为 Integer.MAX_VALUE - 8
		if (newCapacity - MAX_ARRAY_SIZE > 0)
			newCapacity = hugeCapacity(minCapacity);
		elementData = Arrays.copyOf(elementData, newCapacity);
	}
	private static int hugeCapacity(int minCapacity) {
		if (minCapacity < 0) throw new OutOfMemoryError();
		return (minCapacity > MAX_ARRAY_SIZE) ? Integer.MAX_VALUE : MAX_ARRAY_SIZE;
	}

	// 该方法给用户调用，直接扩容指定容量
	// 场景：在知道容量且 add 大量元素时，进行一次固定容量扩容，以减少扩容次数，提高性能
	public void ensureCapacity(int minCapacity) {
		int minExpand = (elementData != DEFAULTCAPACITY_EMPTY_ELEMENTDATA) ? 0 : DEFAULT_CAPACITY;
		if (minCapacity > minExpand) {ensureExplicitCapacity(minCapacity);}
	}
}

AbstractSequentialList 提供了一套基于顺序访问的接口，AbstractSequentialList 提供的方法基本上都是通过 ListIterator 实现。LinkedList 不支持随机访问，不建议使用 for 进行遍历，而是使用迭代遍历（foreach 会转换为迭代器形式）

LinkedList 实现了 Deque 接口，可以作为队列、栈、双端队列应用

# ArrayList/LinkedList 区别

  • 数据结构：ArrayList 是动态数组，LinkedList 是双向链表；有序（插入序），都支持空值和重复值
  • 线程安全：都是非线程安全
  • 时间复杂度：
    • 查找：ArrayList 通过索引快速查找，O(1)；LinkedList 只能通过指针遍历查找，O(n)
    • 插入/删除：ArrayList 可能涉及元素移动，O(n)；LinkedList 不涉及元素移动，O(1)
  • 快速随机访问：ArrayList 支持快速随机访问，LinkedList 不支持
  • 内存空间占用：ArrayList 在动态数组结尾预留一定的容量空间；LinkedList 的元素节点需要额外存储直接后继与前驱节点的指针引用

# Set：三者区别

  • 线程安全：HashSet、LinkedHashSet、TreeSet 都是非线程安全
  • 数据结构：
    • HashSet 基于哈希表（HashMap）实现，插入和取出元素不保证 FIFO，无序；
    • LinkedHashSet 基于链表和哈希表（LinkedHashMap）实现，插入和取出保证 FIFO，有序（插入序）；
    • TreeSet 基于红黑树（TreeMap）实现，元素有序（非插入序），以自然排序和定制排序实现
  • 快速随机访问：三者都不支持快速随机访问，通过迭代器遍历

# Queue - Deque

  • Queue 是单端队列，在队首删除元素，队尾插入元素，遵循先进先出 FIFO 原则
  • Deque 是双端队列，在队列两端均可插入或删除元素，也提供 push()/pop() 入栈出栈方法模拟栈

由于队列容量不足导致队列操作失败，有两种处理方式：抛出异常；返回特殊值（高亮部分）

# Deque：ArrayDeque

// 循环动态数组和双指针
public class ArrayDeque<E> extends AbstractCollection<E> implements Deque<E>, Cloneable, Serializable {
	transient Object[] elements; // 大小总为 2 的幂次方
	transient int head; // 指向双端队列头元素
	transient int tail; // 指向双端队列尾元素的下一位
	private static final int MIN_INITIAL_CAPACITY = 8; // 最小初始化容量
	
	public ArrayDeque() {elements = new Object[16];}
	public ArrayDeque(int numElements) {allocateElements(numElements);} // 初始化大小为8，或大于numElements的最小2的幂次方
	public ArrayDeque(Collection<? extends E> c) { // 初始化大小为大于集合大小的最小 2 的幂次方
		allocateElements(c.size());
		addAll(c);
	}
	private void allocateElements(int numElements) {elements = new Object[calculateSize(numElements)];}
	private static int calculateSize(int numElements) {
		int initialCapacity = MIN_INITIAL_CAPACITY;
		if (numElements >= initialCapacity) {
			initialCapacity = numElements;
			// 无符号右移再进行按位或操作，就是将其低位全部补成1，然后再自加加一次，就是再向前进一位。
			// 这样就能得到其最小的2次幂。之所以需要最多移16位，是为了能够处理大于2^16次方数
			initialCapacity |= (initialCapacity >>>  1);
			initialCapacity |= (initialCapacity >>>  2);
			initialCapacity |= (initialCapacity >>>  4);
			initialCapacity |= (initialCapacity >>>  8);
			initialCapacity |= (initialCapacity >>> 16);
			initialCapacity++;
			// 如果初始值在int最大值2^31-1和2^30之间，进行一系列移位操作后将得到int最大值，再加1，则溢出变成负数，所以需要检测临界值，然后再右移1位
			if (initialCapacity < 0)   // Too many elements, must back off
				initialCapacity >>>= 1;// Good luck allocating 2 ^ 30 elements
		}
		return initialCapacity;
	}
	public void addFirst(E e) {
		if (e == null) throw new NullPointerException(); // ArrayDeque 不支持 NULL 值
		elements[head = (head - 1) & (elements.length - 1)] = e;
		if (head == tail) doubleCapacity(); // 队列满，扩容
	}
	public void addLast(E e) {
		if (e == null) throw new NullPointerException();
		elements[tail] = e;
		if ( (tail = (tail + 1) & (elements.length - 1)) == head) doubleCapacity();
	}
	private void doubleCapacity() {
		assert head == tail;
		int p = head;
		int n = elements.length;
		int r = n - p; // number of elements to the right of p
		int newCapacity = n << 1; // 2 的幂次方扩容
		if (newCapacity < 0) throw new IllegalStateException("Sorry, deque too big");
		Object[] a = new Object[newCapacity];
		System.arraycopy(elements, p, a, 0, r);
		System.arraycopy(elements, 0, a, r, p);
		elements = a;
		head = 0;
		tail = n;
	}
}

# Queue：PriorityQueue

PriorityQueue 是基于优先堆（二叉堆）的一个无界队列，底层采用动态数组实现，入队时通过 Comparable/Comparator 进行元素排序，出队时优先级高的元素先出队

PriorityQueue 是非线程安全的，且不支持存储 NULL 和 non-comparable 的对象

public class PriorityQueue<E> extends AbstractQueue<E> implements java.io.Serializable {
	private static final int DEFAULT_INITIAL_CAPACITY = 11; // 默认初始容量
	transient Object[] queue;
	private final Comparator<? super E> comparator;
	public PriorityQueue(int initialCapacity, Comparator<? super E> comparator) {
		if (initialCapacity < 1) throw new IllegalArgumentException();
		this.queue = new Object[initialCapacity];
		this.comparator = comparator;
	}
	private void grow(int minCapacity) { // 扩容
		int oldCapacity = queue.length;
        int newCapacity = oldCapacity + ((oldCapacity < 64) ? (oldCapacity + 2) : (oldCapacity >> 1));
        if (newCapacity - MAX_ARRAY_SIZE > 0) newCapacity = hugeCapacity(minCapacity);
        queue = Arrays.copyOf(queue, newCapacity);
    }
}

# Map：HashMap

// HashMap：底层结构（散列桶）：数组、链表，链表是为解决哈希冲突
public class HashMap<K,V> extends AbstractMap<K,V> implements Map<K,V>, Cloneable, Serializable {
	static final int DEFAULT_INITIAL_CAPACITY = 1 << 4; // 默认初始容量 16
	static final int MAXIMUM_CAPACITY = 1 << 30; // 最大容量 230
	static final float DEFAULT_LOAD_FACTOR = 0.75f; // 默认负载因子
	static final int TREEIFY_THRESHOLD = 8; // 默认树化阈值，链表长度超过 8，则转为红黑树
	static final int UNTREEIFY_THRESHOLD = 6; // 默认链表阈值，链表长度低于 6，则把红黑树转为链表
	static final int MIN_TREEIFY_CAPACITY = 64; // 扩容临界值，当桶数组容量小于该值时，优先进行扩容，而不是树化
	static class Node<K,V> implements Map.Entry<K,V> {
		final int hash; // key hashcode
		final K key;
		V value;
		Node<K,V> next; // 单链表结构，存储下一个 k-v
		Node(int hash, K key, V value, Node<K,V> next) {
			this.hash = hash;
			this.key = key;
			this.value = value;
			this.next = next;
		}
	}
	transient Node<K,V>[] table; // 链表数组（称为桶），数组中的元素是 Node 的单链表
	transient int size;
	int threshold; // 扩容阈值，初始容量 * 负载因子
	final float loadFactor; // 负载因子，与扩容机制有关
	public HashMap(int initialCapacity, float loadFactor) { // 该构造器没有为数组 table 分配内存空间，在 put 操作时才分配
		if (initialCapacity < 0) throw new IllegalArgumentException("Illegal initial capacity: " + initialCapacity);
		if (initialCapacity > MAXIMUM_CAPACITY) initialCapacity = MAXIMUM_CAPACITY;
		if (loadFactor <= 0 || Float.isNaN(loadFactor)) throw new IllegalArgumentException("Illegal load factor: " + loadFactor);
		this.loadFactor = loadFactor;
		this.threshold = tableSizeFor(initialCapacity);
	}
	// 对 Key HashCode 计算 Hash 值，通过 Hash 值计算数组下标：(n - 1) & Hash，n 为数组大小
	static final int hash(Object key) {
		int h;
		return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
	}
}

HashMap put 操作：通过扰动函数（减少碰撞）对 Key 计算 HashCode 值，然后再计算数组下标，若无碰撞则直接放入桶，否则挂载链表后面；如果链表长度超过阈值则转成红黑树，链表长度低于 6，就把红黑树转回链表；如果桶满了(容量16 * 负载因子0.75)，就需要扩容 2 倍后重排（rehashing）

引入红黑树就是为了查找数据快，解决链表查询过深的问题

扩容：创建一个新的 Node 空数组，长度是原数组 2 倍，遍历原 Node 数组，把所有的 Node 重新 Hash 到新数组

# Map：Hashtable（少用）

  1. Hashtable 在数据结构上与 HashMap 基本一致，但是线程安全的，由 synchronized 修饰。
  2. HashMap 可以存储 null 的 key 和 value，但 Hashtable 不支持
  3. Hashtable 默认的初始大小为 11，之后每次扩充，容量变为原来的 2n+1
  4. Hashtable 只有链表和数组，没有将链表转为红黑树的机制

# 接口：RandmoAccess

  • RandomAccess 是一个标志接口，实现该接口的集合是支持快速随机访问策略
  • 用处是当实现某些算法时，会判断当前类是否实现了 RandomAccess 接口，会选择不同的高效遍历算法
  • Random Access List(随机访问列表)如 ArrayList 实现此接口，Sequence Access List(顺序访问列表)如 LinkedList 不实现此接口
  • 随机访问列表使用循环遍历，顺序访问列表使用迭代器遍历

场景：将操作随机访问列表的最好算法应用到顺序访问列表时，会产生二次项行为。建议一般的列表算法检查给定的列表是否 instanceof 这个接口，防止在顺序访问列表时使用较差的算法

public static <T> int binarySearch(List<? extends Comparable<? super T>> list, T key) {
    if (list instanceof RandomAccess || list.size()<BINARYSEARCH_THRESHOLD)
        return Collections.indexedBinarySearch(list, key);
    else
        return Collections.iteratorBinarySearch(list, key);
}
// 二分查找算法：随机访问列表
private static <T> int indexedBinarySearch(List<? extends Comparable<? super T>> list, T key) {
    int low = 0;
    int high = list.size()-1;

    while (low <= high) {
        int mid = (low + high) >>> 1;
        Comparable<? super T> midVal = list.get(mid); // 索引获取指定位置数据
        int cmp = midVal.compareTo(key);

        if (cmp < 0)
            low = mid + 1;
        else if (cmp > 0)
            high = mid - 1;
        else
            return mid; // key found
    }
    return -(low + 1);  // key not found
}
// 二分查找算法：顺序访问列表
private static <T> int iteratorBinarySearch(List<? extends Comparable<? super T>> list, T key) {
    int low = 0;
    int high = list.size()-1;
    ListIterator<? extends Comparable<? super T>> i = list.listIterator();

    while (low <= high) {
        int mid = (low + high) >>> 1;
        Comparable<? super T> midVal = get(i, mid); // 迭代获取指定位置数据
        int cmp = midVal.compareTo(key);

        if (cmp < 0)
            low = mid + 1;
        else if (cmp > 0)
            high = mid - 1;
        else
            return mid; // key found
    }
    return -(low + 1);  // key not found
}

# 接口：Cloneable

Cloneable 是个标记接口，实现该接口并重写 Object.clone() 方法，类实例才能被克隆，否则直接调用 Object.clone() 方法会抛出异常 CloneNotSupportedException

/**
 * Object.clone() 由 native 修饰的空方法，调用该方法先判断类是否实现 Clonable 接口，未实现则抛出异常
 * abstract 修饰的空方法需要子类实现，而 native 修饰的空方法在 JVM 中都有同名的实现体，对返回类型，异常控制等都没有约束
 */
protected native Object clone() throws CloneNotSupportedException;

  2. 深克隆：实现方法：

# 接口：Comparable 自然排序

java.lang.Comparable 自然排序：实现该接口的类实例间能够进行自然排序

public interface Comparable<T> {
	public int compareTo(T o);
}
1
2
3
  1. 重写 compareTo 方法时应该注意 o.compareTo(null) 情况，即使 o.equals(null) = false，compareTo 方法也应该主动抛出一个空指针异常 NullPointerException
  2. 重写 compareTo 方法时一般要求 o.compareTo(e) 的结果要和 o.equals(e) 保持一致，这样在使用如 SortedSet 根据类的自然排序进行排序的有序集合 时可以保证顺序和想象中一致
  3. 实现了 Comparable 接口的 List 或数组元素可以使用 Collections.sort()/Arrays.sort() 方法进行自然排序
  4. 实现了 Comparable 接口的类实例才能直接被用作 SortedMap/SortedSet key，要不然得指定 Comparator 定制排序
